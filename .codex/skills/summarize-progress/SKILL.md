# Summarize Progress

## Purpose

Use this skill to produce structured summaries after meaningful implementation or investigation work.

## When to use

Use this skill when:
- a task step has completed
- a checkpoint summary is needed
- a handoff note is needed
- progress needs to be recorded in task artifacts

## Workflow

1. Identify what changed.
2. Identify why the change was made.
3. List files touched or sources examined.
4. Record validation performed.
5. Record unresolved risks or open questions.
6. Suggest the next step.

## Output shape

Prefer this structure:
- completed work
- rationale
- files or sources involved
- validation performed
- unresolved issues
- recommended next step

## Constraints

- Be concrete.
- Avoid vague status language.
- Distinguish confirmed work from future intentions.
