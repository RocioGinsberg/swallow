# Phase 1 Draft

## Direction

Phase 1 is intended to improve the quality and extensibility of the MVP without turning the system into a broad platform too early.

## Likely areas of work

- improve retrieval quality
- introduce better chunking and metadata handling
- refine the harness runtime
- add a second executor
- refine orchestration policies
- introduce richer capability packs
- add validators and stronger execution policies
- enrich artifact indexing and memory
- strengthen validation and test coverage

## Likely non-goals

- broad provider-routing layer
- full desktop UI
- large source-ingestion matrix
- large plugin marketplace support

## Entry condition

Phase 1 should start only after the Phase 0 CLI loop is stable and useful on real tasks.

## Notes

This is a directional draft, not a locked implementation contract.
